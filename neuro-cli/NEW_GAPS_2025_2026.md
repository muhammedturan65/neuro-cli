## NEW GAP LIST (Prioritized)

### P0 — Critical (Competitive Must-Haves, Ship in Next Sprint)

#### GAP-27: Sub-Agent Spawning with Scoped Tool Access
- **Who has it**: Claude Code (Task tool + YAML subagents), Goose (subagents), Codex CLI (task delegation)
- **What it is**: Parent orchestrator spawns child agents with: custom system prompt, restricted tool set (allowlist/denylist), scoped file access, and result handoff contract. Claude Code subagents are defined in `claude/agents/` with YAML frontmatter specifying `tools:` allowlist and `prompt:` 
- **NeuroCLI gap**: Has `parallel-agents.ts` and `team.ts` (inter-agent messaging) and `custom-agents.ts` but lacks the **Task delegation protocol** — a parent agent calling a sub-agent with scoped capabilities and getting structured results back. Current agents are peer-level, not hierarchical.
- **Impact**: Multi-agent orchestration is the #1 differentiator in 2026. Without scoped sub-agents, NeuroCLI can't safely decompose complex tasks.

#### GAP-28: Agent Client Protocol (ACP) Support
- **Who has it**: OpenHands, VS Code (unified agent experience), JetBrains, Zed
- **What it is**: Open JSON-RPC standard (like LSP but for AI agents) that lets any editor/IDE communicate with any coding agent. Standardizes: file operations, diff application, command execution, status reporting. Introduced Aug 2025 by Zed.
- **NeuroCLI gap**: No ACP server implementation. NeuroCLI can only be used standalone in terminal, not as a backend for VS Code, JetBrains, Zed, or other editors.
- **Impact**: Without ACP, NeuroCLI is locked out of the IDE ecosystem. All major competitors are adding ACP support.

#### GAP-29: OS-Level Sandbox with Network Isolation
- **Who has it**: Codex CLI (OS-level sandbox on Windows/Mac/Linux with filesystem + network controls), Docker Sandboxes (for Claude Code, Codex, etc.)
- **What it is**: Sandbox that restricts both filesystem access AND network access. Codex CLI runs commands in an OS-level sandbox. Docker Sandboxes product from Docker isolates local filesystem but also secures network access and API credentials.
- **NeuroCLI gap**: `sandbox.ts` provides file isolation only (denied patterns, backup/undo). No network isolation, no OS-level sandboxing, no Docker integration.
- **Impact**: Security incident stories (rm -rf ~/) are mainstream. Enterprise adoption requires proper sandboxing.

#### GAP-30: Spec-Driven Development Pipeline
- **Who has it**: Kiro (AWS) — spec-first development, cc-sdd (community Claude Code spec-driven)
- **What it is**: Generate structured specifications (requirements, design docs, implementation plans) from natural language prompts BEFORE writing code. Specs become the source of truth. Kiro's pipeline: Prompt → Requirements Spec → Design Doc → Implementation → Verification against spec.
- **NeuroCLI gap**: No spec generation or spec-driven workflow. NeuroCLI goes directly from prompt to code.
- **Impact**: Spec-driven development is the 2026 answer to "vibe coding" quality problems. It's becoming the enterprise standard.

#### GAP-31: Hooks with LLM Evaluator Support
- **Who has it**: Claude Code (hooks with PreToolCall, PostToolCall, Notification, Stop handlers — including LLM evaluators)
- **What it is**: Lifecycle hooks that can invoke either shell commands OR LLM-based evaluators. An LLM evaluator hook can analyze the proposed action and approve/deny/modify it. E.g., a PreToolCall hook that uses a cheap model to verify the edit won't break tests.
- **NeuroCLI gap**: `hooks.ts` has 20 lifecycle events but they only invoke shell commands. No LLM-based evaluator hooks.
- **Impact**: LLM evaluator hooks are the key to safe autonomous operation — they're the "smart monitor" that makes Auto Mode viable without human oversight.

#### GAP-32: MCP Apps / Interactive Tool UI Extensions
- **Who has it**: Goose (pioneered MCP Apps), MCP official (now an official MCP extension as of Jan 2026)
- **What it is**: MCP tools can return interactive HTML/JS UI components that render directly in the agent conversation. Buttons, forms, visualizations, data tables. Goose renders these in Desktop; the standard defines the protocol.
- **NeuroCLI gap**: MCP client only supports standard tool calls (text in/text out). No support for MCP Apps extension or interactive UI rendering.
- **Impact**: MCP Apps is the future of agent-tool interaction. Without it, NeuroCLI's tools are limited to text exchanges.

### P1 — High (Significant Competitive Advantage, Ship in 1-2 Sprints)

#### GAP-33: Multi-Model Orchestrator/Worker Pattern (Cost Stratification)
- **Who has it**: Claude Code (orchestrator model directs cheaper sub-agents), Aider (Architect mode: expensive planning model + cheap editing model)
- **What it is**: Use an expensive/capable model as orchestrator to plan, then delegate execution to cheaper/faster models. Aider's Architect mode cuts session costs by ~60% by using a planning model only for strategy and a cheap model for edits.
- **NeuroCLI gap**: Model router selects ONE model per task. No orchestrator/worker split where different models handle different phases.
- **Impact**: Cost optimization is the #2 concern after capability. Multi-model stratification can cut costs 50-70%.

#### GAP-34: Auto Mode with Smart Monitor (Not Just Auto-Approve)
- **Who has it**: Claude Code (Auto Mode uses a monitor LLM to approve/deny each action)
- **What it is**: Instead of blanket auto-approve (NeuroCLI's "auto" mode) or blanket deny (manual mode), a lightweight monitor LLM evaluates each proposed action in real-time and decides whether to approve, deny, or ask the user. It's a safety layer between full-auto and manual.
- **NeuroCLI gap**: Auto mode auto-approves "safe" operations based on static rules. Yolo auto-approves everything. No dynamic LLM-based evaluation of actions.
- **Impact**: Smart monitoring enables safe autonomous operation without constant human oversight. It's the bridge between manual and yolo.

#### GAP-35: Rubric-Based Outcome Grading
- **Who has it**: Claude Code Managed Agents (Outcomes feature, June 2026)
- **What it is**: Define rubrics describing what success looks like. A separate evaluator (in its own context window, isolated from agent reasoning) grades the output. The agent revises until it meets the rubric. Quality assurance loop.
- **NeuroCLI gap**: No outcome evaluation system. No rubric definition, no separate grader, no revision loop.
- **Impact**: Without outcome grading, there's no automated quality gate for autonomous agent runs.

#### GAP-36: OpenTelemetry / Observability Integration
- **Who has it**: Gemini CLI (OpenTelemetry-based observability, Dynatrace integration), ai-observer (unified local observability)
- **What it is**: Standard OpenTelemetry traces and metrics for AI agent operations: token usage per request, latency, error rates, cost tracking, session activity. Exportable to Datadog, Dynatrace, Jaeger, etc.
- **NeuroCLI gap**: `telemetry.ts` exists but is custom/internal. No OpenTelemetry integration, no standard observability export, no tracing.
- **Impact**: Enterprise teams need observability in their existing monitoring stack. Custom telemetry doesn't integrate.

#### GAP-37: Auto-Compact with Model-Aware Context Management
- **Who has it**: OpenCode (auto-compact automatically summarizes conversation when approaching context window limit)
- **What it is**: Automatic, proactive context compaction that's aware of the current model's exact context window size. When usage approaches the limit, it automatically triggers summarization — not waiting for the user to notice and run `/compact`.
- **NeuroCLI gap**: Has 5-layer compaction but it's manual (`/compact`). No automatic trigger based on context window proximity. Not model-aware (different models have different context sizes).
- **Impact**: Auto-compact prevents context overflow failures that derail long coding sessions.

#### GAP-38: Interactive Terminal Features (OSC 52, TUI, Embedded Commands)
- **Who has it**: Gemini CLI (OSC 52 paste support, run interactive commands like vim/top inside CLI), OpenCode (native TUI with split panes)
- **What it is**: Rich terminal integration: OSC 52 clipboard support for remote sessions, ability to launch and interact with TUI apps (vim, top, htop) within the agent's terminal session, native TUI with split panes, syntax highlighting, inline diffs with color.
- **NeuroCLI gap**: Basic terminal output with 4 themes. No OSC 52 support, no embedded interactive commands, no split-pane TUI, no terminal-native syntax highlighting.
- **Impact**: Rich terminal UX is what separates a "tool" from a "development environment." CLI agents are becoming IDE replacements.

#### GAP-39: Multi-Session Parallel Agents with Independent State
- **Who has it**: OpenCode (multi-session: start multiple agents in parallel on different tasks with independent context)
- **What it is**: Run multiple independent agent sessions simultaneously, each with its own context, working on different tasks. Not just parallel tool calls within one session, but fully independent sessions.
- **NeuroCLI gap**: `parallel-agents.ts` supports parallel agent execution within one session. Not multiple independent sessions with separate context windows.
- **Impact**: Developers work on multiple things at once. Multi-session is essential for productivity.

#### GAP-40: Git Worktree Integration for Parallel Development
- **Who has it**: Referenced as a key CLI agent advantage (agenticoding.ai)
- **What it is**: Agents operating in different git worktrees simultaneously, enabling truly parallel development without merge conflicts during agent execution.
- **NeuroCLI gap**: Git checkpoint and auto-commit exist, but no git worktree awareness or multi-worktree agent coordination.
- **Impact**: Parallel agent work requires worktree isolation to avoid file conflicts.

### P2 — Medium (Nice to Have, Emerging Patterns)

#### GAP-41: Scheduled Tasks / Routines / Cron-Style Agent Scheduling
- **Who has it**: Claude Code (routines for scheduled agent tasks, cron-style scheduling), Tembo (scheduled tasks)
- **What it is**: Define recurring agent tasks: "Run code review every morning at 9am", "Check test coverage daily", "Triage new issues every hour". Fire-and-forget background agents on schedule.
- **NeuroCLI gap**: `scheduled-tasks.ts` exists but unclear if it supports cron-style scheduling or just one-time delayed tasks. Need to verify and extend.
- **Impact**: Routines turn the CLI from an interactive tool into a persistent team member.

#### GAP-42: Dreaming / Background Memory Curation
- **Who has it**: Claude Code Managed Agents (dreaming pass, June 2026)
- **What it is**: Between sessions, the agent runs a "dreaming pass" — reviewing, reorganizing, and consolidating its memory. It identifies patterns, updates its understanding of the codebase, and primes itself for future tasks.
- **NeuroCLI gap**: No background memory curation. Memory is only updated during active sessions.
- **Impact**: Dreaming enables the agent to improve between sessions, building deeper codebase understanding over time.

#### GAP-43: Steering Files / Structured Agent Constraints
- **Who has it**: Kiro (steering files that constrain and guide the agent)
- **What it is**: More structured than NEURO.md, steering files define hard constraints, preferred patterns, coding standards, and project-specific rules that the agent MUST follow. Violation = hard error, not just ignored context.
- **NeuroCLI gap**: NEURO.md provides context but isn't enforced. No concept of "hard constraints" that the agent must satisfy.
- **Impact**: Steering files give teams control over agent behavior in enterprise settings.

#### GAP-44: MCP 2026 Stateless Protocol + Server Discovery
- **Who has it**: MCP 2026-07-28 spec (stateless core, .well-known metadata)
- **What it is**: MCP goes stateless — no session handshake, no session header. Standard metadata format via `.well-known` for server capability discovery without live connection. Horizontal scaling for MCP servers.
- **NeuroCLI gap**: MCP client uses current session-based protocol. No support for stateless mode or server discovery.
- **Impact**: Stateless MCP is the future. Session-based MCP can't scale for enterprise.

#### GAP-45: Cost Tracking Dashboard / Real-Time Spend Visualization
- **Who has it**: TokenShift (on-device), ai-observer (unified dashboard), PointFive
- **What it is**: Real-time cost tracking dashboard showing: spend per model, spend per session, spend per project, cache hit rates, token usage trends, budget alerts with visualizations.
- **NeuroCLI gap**: `spending-warnings.ts` tracks spending with limits but no dashboard, no trends, no per-project breakdown, no visualization.
- **Impact**: Cost visibility drives cost optimization. You can't cut what you can't see.

#### GAP-46: Cloud Agent Execution / Hybrid Local+Cloud
- **Who has it**: Codex CLI (cloud tasks), OpenHands (cloud sandbox), Tembo (cloud agents)
- **What it is**: Run agent tasks in the cloud with preloaded repos, isolated environments, and no local resource consumption. Hybrid: simple tasks local, heavy tasks cloud.
- **NeuroCLI gap**: Local-only execution. No cloud execution, no hybrid routing.
- **Impact**: Cloud execution enables parallel work without local resource constraints.

#### GAP-47: Prompt Compression (SCOPE Approach)
- **Who has it**: SCOPE (Zhang et al. 2025), Headroom, various compression tools
- **What it is**: Use a generative summarization model to compress prompts/context before sending to the main model. Achieves 50-96% effective cost reduction combined with provider KV caching.
- **NeuroCLI gap**: Prompt cache uses SHA-256 matching. Context compaction uses summarization. But no dedicated prompt compression layer that pre-processes input before sending to the model.
- **Impact**: Prompt compression is a different approach from caching — it reduces the size of every request, not just repeated ones.

#### GAP-48: Voice Input for Coding
- **Who has it**: Aider (voice input for coding requests)
- **What it is**: Speak coding instructions instead of typing. "Refactor this function to use async/await" — said aloud, transcribed, and executed.
- **NeuroCLI gap**: `voice.ts` exists but unclear scope. Need to verify if it supports voice input (STT) or only output (TTS).
- **Impact**: Voice input is an accessibility and convenience feature, especially for long instructions.

#### GAP-49: Live Web Search Integration During Coding
- **Who has it**: Codex CLI (live search feature)
- **What it is**: Agent can search the web in real-time during coding tasks: look up API docs, find Stack Overflow answers, check latest library versions.
- **NeuroCLI gap**: Has `web.ts` tool but unclear if it supports real-time web search or just URL reading.
- **Impact**: Code doesn't exist in a vacuum. Agents need current information.

#### GAP-50: Image/Vision Token Compression for Multimodal
- **Who has it**: Various tools (59-70% token cost reduction for image-heavy workloads)
- **What it is**: Compress image tokens before sending to multimodal models. Critical when using vision capabilities (screenshots, diagrams, UI mockups).
- **NeuroCLI gap**: `multimodal.ts` exists but no image token compression.
- **Impact**: Vision is expensive. Without compression, multimodal features are cost-prohibitive.

### P3 — Low (Experimental / Future-Looking)

#### GAP-51: Agent-to-Agent Communication Protocol (A2A)
- **Who has it**: Google's A2A protocol, IBM/Linux Foundation Agent Commerce Protocol
- **What it is**: Standardized protocol for inter-agent communication: discovery, negotiation, task handoff, result exchange between agents from different vendors/frameworks.
- **NeuroCLI gap**: No A2A support. Inter-agent messaging (`team.ts`) is internal only.
- **Impact**: A2A is early-stage but will matter as multi-vendor agent ecosystems emerge.

#### GAP-52: Computer Use / UI Automation from CLI
- **Who has it**: Codex CLI (computer control capability)
- **What it is**: Agent can control the computer's UI: click buttons, fill forms, navigate apps — not just terminal commands.
- **NeuroCLI gap**: No computer use capability. Terminal and file operations only.
- **Impact**: Enables browser testing, GUI interaction, and desktop app automation from CLI.

#### GAP-53: Rust-Based Performance Core
- **Who has it**: Codex CLI (95.6% Rust), OpenDev (Rust)
- **What it is**: Core agent logic in Rust for speed: faster file operations, lower memory usage, native binary distribution.
- **NeuroCLI gap**: TypeScript/Node.js throughout.
- **Impact**: Rust core would improve startup time, memory usage, and enable native binary distribution. But TS ecosystem is richer.

#### GAP-54: LEANN-style Personal Vector DB / Semantic Codebase Search
- **Who has it**: LEANN (MLsys2026), various RAG approaches
- **What it is**: Local vector database for semantic search over entire file system and codebase. Not just grep — semantic understanding.
- **Note**: Anthropic REMOVED vector search from Claude Code in May 2025 — ripgrep beat it for code search. But semantic search still matters for documentation, design docs, and non-code knowledge.
- **NeuroCLI gap**: `memory.ts` and `repo-map.ts` exist but no vector database or semantic search.
- **Impact**: Debated. For code: grep wins. For docs/knowledge: semantic search still valuable.

#### GAP-55: Parallel-Agent Reinforcement Learning Orchestration
- **Who has it**: Claude Code K2.6 (up to 100 sub-agents, 1,500 tool calls in parallel via RL)
- **What it is**: Advanced orchestration using RL to coordinate large numbers of parallel agents optimally.
- **NeuroCLI gap**: Basic parallel execution. No RL-based coordination.
- **Impact**: Cutting-edge research, not yet mainstream. Future-proofing.

#### GAP-56: Cross-Tool Project Instruction Compatibility (AGENTS.md / CLAUDE.md)
- **Who has it**: Codex CLI (AGENTS.md), Claude Code (CLAUDE.md), Kiro (steering files)
- **What it is**: Standardized project instruction file that works across tools. `cp CLAUDE.md AGENTS.md` — but ideally a universal format.
- **NeuroCLI gap**: Uses NEURO.md. No compatibility layer or migration path for CLAUDE.md/AGENTS.md.
- **Impact**: Developers use multiple tools. Project instructions should be portable.

#### GAP-57: Dependency Graph Analysis for Context
- **Who has it**: Implied by advanced context engineering papers (OpenDev)
- **What it is**: Build and traverse the project's dependency graph (imports, requires, package dependencies) to understand impact of changes. Not just AST parsing — full dependency resolution.
- **NeuroCLI gap**: `tree-sitter.ts` provides AST parsing. No dependency graph construction or traversal.
- **Impact**: Better understanding of change impact → fewer broken builds.

#### GAP-58: AI-Assisted Git Conflict Resolution
- **Who has it**: Emerging feature in multi-agent workflows
- **What it is**: When parallel agents modify the same files, AI-assisted merge conflict resolution that understands the intent of both changes.
- **NeuroCLI gap**: No conflict resolution. Git checkpoint exists but doesn't handle conflicts from parallel agents.
- **Impact**: Critical for multi-agent parallel workflows.

#### GAP-59: Standardized Test Generation with Coverage Analysis
- **Who has it**: Aider (auto test+fix loop), Amazon Q (test generation), various tools
- **What it is**: Not just running tests (NeuroCLI has this) but: (1) generating test files from source, (2) measuring coverage, (3) auto-fixing failing tests, (4) TDD workflow where tests are written first.
- **NeuroCLI gap**: `testing.ts` runs tests and detects frameworks. `generateTest()` exists but no coverage analysis, no TDD workflow, no auto-fix loop.
- **Impact**: Test generation + auto-fix is a complete quality loop, not just a test runner.

#### GAP-60: MCP Registry / Server Discovery via .well-known
- **Who has it**: MCP 2026 roadmap (standard metadata format)
- **What it is**: Standard `.well-known/mcp` endpoint that describes MCP server capabilities without requiring a live connection. Enables server registries, auto-discovery, and cataloging.
- **NeuroCLI gap**: No server discovery. MCP servers must be manually added via `neuro mcp add`.
- **Impact**: Auto-discovery would make MCP setup zero-config for common tools.

## Top 5 Recommended Implementation Order

1. **GAP-27 (Sub-Agent Spawning)** — The most impactful gap. Multi-agent is 2026's defining feature.
2. **GAP-29 (OS-Level Sandbox)** — Security is table-stakes for enterprise. Docker sandbox integration.
3. **GAP-28 (ACP Support)** — Interoperability with IDEs expands NeuroCLI's reach dramatically.
4. **GAP-33 (Multi-Model Orchestrator/Worker)** — Cost stratification via architect/coder model split.
5. **GAP-31 (LLM Evaluator Hooks)** — Enables safe Auto Mode, which enables autonomous operation.
